# Emit.Persistence.PostgreSQL

PostgreSQL persistence provider for Emit using Entity Framework Core. Stores outbox entries in PostgreSQL tables with atomic sequence generation and full EF Core integration.

## Installation

```bash
dotnet add package Emit.Persistence.PostgreSQL
```

## Quick Start

```csharp
services.AddEmit(builder =>
{
    builder.UsePostgreSql((sp, options) =>
    {
        options.ConnectionString = "Host=localhost;Database=myapp;Username=postgres;Password=secret";
        options.TableName = "outbox";
    });

    builder.AddKafka((sp, kafka) => { /* ... */ });
});
```

## Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| `ConnectionString` | Required | PostgreSQL connection string |
| `TableName` | `"outbox"` | Outbox table name |
| `LeaseTableName` | `"outbox_lease"` | Worker lease table name |
| `SequenceTableName` | `"outbox_sequences"` | Sequence counter table name |
| `RetentionPeriod` | 7 days | How long to keep completed entries |
| `MaxAttemptsPerEntry` | 10 | Maximum attempt records to store per entry |

## Table Schema

### Outbox Table

```sql
CREATE TABLE outbox (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    provider_id VARCHAR(100) NOT NULL,
    registration_key VARCHAR(200) NOT NULL,
    group_key VARCHAR(500) NOT NULL,
    sequence BIGINT NOT NULL GENERATED BY DEFAULT AS IDENTITY,
    status VARCHAR(50) NOT NULL,
    enqueued_at TIMESTAMP WITH TIME ZONE NOT NULL,
    completed_at TIMESTAMP WITH TIME ZONE,
    retry_count INTEGER DEFAULT 0,
    last_attempted_at TIMESTAMP WITH TIME ZONE,
    latest_error TEXT,
    attempts JSONB,
    payload BYTEA NOT NULL,
    properties JSONB,
    correlation_id VARCHAR(200)
);
```

### Sequence Table

Used for atomic sequence number generation per group:

```sql
CREATE TABLE outbox_sequences (
    group_key VARCHAR(500) PRIMARY KEY,
    sequence BIGINT DEFAULT 0
);
```

### Lease Table

Used for single-worker coordination:

```sql
CREATE TABLE outbox_lease (
    id VARCHAR(50) PRIMARY KEY,
    worker_id VARCHAR(200),
    lease_until TIMESTAMP WITH TIME ZONE NOT NULL
);
```

## Indexes

The following indexes are created automatically via EF Core migrations:

```sql
-- Unique compound index for ordering within groups
CREATE UNIQUE INDEX ix_outbox_group_key_sequence ON outbox (group_key, sequence);

-- Composite index for efficient group head queries
CREATE INDEX ix_outbox_status_group_key ON outbox (status, group_key);

-- Index for cleanup queries
CREATE INDEX ix_outbox_completed_at ON outbox (completed_at);
```

## Sequence Number Generation

PostgreSQL uses an atomic upsert pattern for sequence number generation:

```sql
INSERT INTO outbox_sequences (group_key, sequence)
VALUES (@groupKey, 1)
ON CONFLICT (group_key)
DO UPDATE SET sequence = outbox_sequences.sequence + 1
RETURNING sequence
```

This approach:
- Guarantees globally ordered sequences within each group across all application replicas
- Is atomic and thread-safe without requiring distributed locks
- Creates the counter row automatically on first enqueue to a group

## Transaction Integration

The PostgreSQL provider integrates with the [Transactional](https://github.com/ChuckNovice/transactional) library:

```csharp
public class OrderService
{
    private readonly IPostgresTransactionManager transactionManager;
    private readonly OrderDbContext dbContext;
    private readonly IProducer<string, OrderCreated> producer;

    public async Task CreateOrderAsync(Order order)
    {
        await using var transaction = await transactionManager.BeginTransactionAsync();

        // Your business data write
        dbContext.Orders.Add(order);
        await dbContext.SaveChangesAsync();

        // Outbox write (same transaction)
        await producer.ProduceAsync("orders", new Message<string, OrderCreated>
        {
            Key = order.Id.ToString(),
            Value = new OrderCreated { OrderId = order.Id }
        });

        // Both commit atomically
        await transaction.CommitAsync();
    }
}
```

### Sharing Transactions with Your DbContext

The outbox repository automatically enlists in the transaction provided via `IPostgresTransactionContext`:

```csharp
// Transaction is shared between your DbContext and the outbox
await dbContext.Database.UseTransactionAsync(transaction.Transaction);
```

## Entity Framework Core Integration

The provider uses EF Core for database operations, which provides:

- **Automatic schema creation**: Tables and indexes are created on startup via `EnsureCreatedAsync`
- **JSONB storage**: Properties and attempts are stored as native PostgreSQL JSONB
- **Value converters**: Automatic conversion for enums, UUIDs, and complex types
- **Bulk operations**: Uses `ExecuteDeleteAsync` for efficient batch cleanup

### Data Types

| Property | PostgreSQL Type | Notes |
|----------|-----------------|-------|
| `Id` | `uuid` | Auto-generated via `gen_random_uuid()` |
| `Sequence` | `bigint` | Identity column |
| `Status` | `varchar(50)` | Stored as string |
| `EnqueuedAt` | `timestamp with time zone` | UTC timestamps |
| `Payload` | `bytea` | Binary MessagePack data |
| `Properties` | `jsonb` | Key-value metadata |
| `Attempts` | `jsonb` | Array of attempt records |

## Cleanup

Completed entries are automatically purged by the cleanup worker based on configuration:

```csharp
// In appsettings.json or configure programmatically
{
  "Emit": {
    "Cleanup": {
      "RetentionPeriod": "7.00:00:00",  // 7 days
      "CleanupInterval": "01:00:00",     // 1 hour
      "BatchSize": 1000
    }
  }
}
```

The cleanup uses EF Core's bulk delete for efficiency:

```csharp
await dbContext.OutboxEntries
    .Where(e => e.Status == OutboxStatus.Completed && e.CompletedAt < cutoffDate)
    .Take(batchSize)
    .ExecuteDeleteAsync();
```

## Troubleshooting

### "Duplicate key violates unique constraint" on sequence

This indicates a race condition in sequence generation. Ensure:
- The sequence table exists and is properly configured
- Transactions are being used correctly
- No manual sequence assignment is occurring

### Slow group head queries

Verify the composite index on `(status, group_key)` exists:

```sql
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'outbox';
```

### Connection pool exhaustion

The provider uses `IDbContextFactory<OutboxDbContext>` which creates new contexts per operation. Ensure your connection pool is sized appropriately:

```csharp
options.ConnectionString = "Host=localhost;Database=myapp;...;Maximum Pool Size=100";
```

### High memory usage during cleanup

Reduce the batch size in cleanup configuration:

```csharp
options.Cleanup.BatchSize = 500; // Lower batch size
```
