# Security Policy

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Use [GitHub Security Advisories](https://github.com/emitdotnet/Emit/security/advisories/new) to report vulnerabilities privately. This ensures the issue is handled confidentially before any public disclosure.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact

### Response Timeline

- **Acknowledgment**: within 48 hours
- **Assessment**: within 7 days
- **Fix target**: within 30 days of confirmed vulnerability

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest pre-release | Yes |
| Older pre-releases | No |

Once Emit reaches a stable `1.0.0` release, this table will be updated with a formal support policy.
