## IEventProducer

Once a topic has been declared with `topic.Producer()`, Emit registers `IEventProducer<TKey, TValue>` in the DI container. Inject it wherever you need to produce messages:

```csharp
public class PizzeriaService(IEventProducer<string, PizzaOrdered> producer)
{
    public async Task PublishAsync(PizzaOrdered pizza, CancellationToken ct)
    {
        await producer.ProduceAsync(
            new EventMessage<string, PizzaOrdered>(pizza.PizzaId.ToString(), pizza),
            ct);
    }
}
```

`IEventProducer` is scoped. Inject it into controllers, endpoint handlers, or any scoped service.

## EventMessage

`EventMessage<TKey, TValue>` carries the key, value, and optional headers:

```csharp
// Key + value only
var msg = new EventMessage<string, PizzaOrdered>(pizza.PizzaId.ToString(), pizza);

// With headers
var msg = new EventMessage<string, PizzaOrdered>(
    pizza.PizzaId.ToString(),
    pizza,
    [new KeyValuePair<string, string>("x-tenant-id", tenantId)]);
```

Headers are passed through to Kafka as-is. Emit also appends its own tracing headers when OpenTelemetry is enabled.

## Outbox mode vs. direct mode

Each producer independently chooses whether to use the outbox or write directly to Kafka. Call `UseOutbox()` on the producer builder to opt in:

```csharp
kafka.Topic<string, PizzaOrdered>("pizzas", topic =>
{
    // ... serializers ...
    topic.Producer(p => p.UseOutbox());  // outbox mode
});

kafka.Topic<string, AnalyticsEvent>("analytics", topic =>
{
    // ... serializers ...
    topic.Producer();  // direct mode
});
```

**Outbox mode** (`p.UseOutbox()`): `ProduceAsync` writes to the database outbox inside the current `IEmitContext.Transaction`. The background worker delivers to Kafka later. Your calling code must begin a transaction first.

**Direct mode** (no `UseOutbox()`): `ProduceAsync` calls the Kafka producer directly and waits for the broker acknowledgement before returning.

Use the outbox when you need atomicity with your database writes. Use direct mode when you want simplicity and can tolerate the dual-write risk or don't have a database write to coordinate with.

:::note
`UseOutbox()` on the producer requires a persistence provider with its own `UseOutbox()` call (e.g. `mongo.UseOutbox()` or `ef.UseOutbox()`). If no persistence provider has enabled the outbox, calling `p.UseOutbox()` throws at registration time.
:::

## Per-producer middleware

Register middleware that runs only for a specific topic's producer:

```csharp
kafka.Topic<string, PizzaOrdered>("pizzas", topic =>
{
    topic.Producer(producer =>
    {
        producer.UseOutbox();
        producer.Use<EnrichmentMiddleware>();
    });
});
```

The middleware receives `SendContext<PizzaOrdered>` and calls `next.InvokeAsync(context)` to continue the pipeline. This is the Leaf level of the [pipeline model](../concepts/pipeline-model/).

## Accessing the message key in middleware

The Kafka key is available via the `KafkaTransportContext<TKey>` payload on the send context:

```csharp
public class LoggingMiddleware<T>(ILogger<LoggingMiddleware<T>> logger)
    : IMiddleware<SendContext<T>>
{
    public Task InvokeAsync(SendContext<T> context, IMiddlewarePipeline<SendContext<T>> next)
    {
        if (context.TryGetPayload<KafkaTransportContext<string>>() is { } kafka)
        {
            logger.LogDebug("Producing with key {Key}", kafka.Key);
        }
        return next.InvokeAsync(context);
    }
}
```

## Produce observers

`IProduceObserver` gives lifecycle hooks without writing middleware:

```csharp
public class ProduceAuditObserver : IProduceObserver
{
    public Task OnProducedAsync<T>(SendContext<T> context) { ... }
    public Task OnProduceErrorAsync<T>(SendContext<T> context, Exception exception) { ... }
}
```

Register at the global level on `EmitBuilder`:

```csharp
emit.AddProduceObserver<ProduceAuditObserver>();
```
