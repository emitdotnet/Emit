---
title: Feature Collection
description: The type-safe feature bag on MessageContext and all built-in features.
---

Every `MessageContext` carries an `IFeatureCollection`, a type-safe dictionary of named capabilities attached to the current message. Middleware and observers use it to share information without knowing about each other, and without coupling to a specific transport.

## Reading a feature

```csharp
var feature = context.Features.Get<IRetryAttemptFeature>();
if (feature is not null)
{
    logger.LogDebug("Attempt {Attempt}", feature.Attempt);
}
```

`Get<T>` returns `null` if the feature is not present. Features are optional by design: a Kafka consumer has `IKeyFeature<TKey>` but a mediator request does not.

## Writing a feature

```csharp
context.Features.Set<IMyCustomFeature>(new MyCustomFeature(value));
```

Later middleware in the same pipeline can read it.

## Built-in features

### IActivityFeature

W3C trace context. Set by the tracing middleware when OpenTelemetry is enabled. Present on all context types.

| Member | Description |
|---|---|
| `TraceParent` | W3C traceparent header (`"00-traceid-spanid-flags"`) |
| `TraceState` | W3C tracestate header |
| `Baggage` | Distributed baggage as key-value pairs |

### IConsumerIdentityFeature

Identifies the handler that will process (or processed) this message. Set before the consumer is invoked; updated by routers after route selection. Present on inbound contexts.

| Member | Description |
|---|---|
| `Identifier` | Consumer type name, or router ID |
| `Kind` | `Direct` or `Router` |
| `ConsumerType` | Resolved handler type |
| `RouteKey` | Matched route key (routers only) |

### IKeyFeature\<TKey\>

The typed message key. `Key` holds the deserialized key of type `TKey`. Present on Kafka inbound and outbound contexts.

### IKeyTypeFeature

The CLR type of the message key, for code that cannot use generics. `KeyType` returns `System.Type`. Present wherever `IKeyFeature<TKey>` is present.

### IMessageSourceFeature

Transport-agnostic source metadata as a string dictionary. Present on inbound contexts.

`Properties` contains transport-specific values. For Kafka, this includes `topic`, `partition`, `offset`, and `provider`. Useful for logging and building DLQ headers without hardcoding transport details.

### IRetryAttemptFeature

The current retry attempt count. `Attempt` is `0` on the initial attempt, `1` on the first retry, and so on. Set by error handling middleware; not present on the initial attempt.

### IHeadersFeature

Transport headers as `IReadOnlyList<KeyValuePair<string, string>>`. Present on Kafka inbound and outbound contexts.

### IRawBytesFeature

The raw serialized bytes of the message. Present on Kafka inbound contexts; used internally by the DLQ sink to forward bytes without re-serializing.

| Member | Description |
|---|---|
| `RawKey` | Serialized key bytes, or `null` |
| `RawValue` | Serialized value bytes, or `null` |

### IProviderIdentifierFeature

The string identifier of the current provider. `ProviderId` returns the provider name (for example, `"kafka"`). Present on all contexts.

### IResponseFeature

For request-response patterns (mediator), carries the response back to the caller. Not present for Kafka consumers.

| Member | Description |
|---|---|
| `HasResponded` | Whether a response has already been set |
| `RespondAsync<TResponse>` | Sets the response and completes the request |

## Feature presence by context type

| Feature | Kafka inbound | Kafka outbound | Mediator |
|---|---|---|---|
| `IActivityFeature` | Yes | Yes | Yes |
| `IConsumerIdentityFeature` | Yes | No | Yes |
| `IKeyFeature<TKey>` | Yes | Yes | No |
| `IKeyTypeFeature` | Yes | Yes | No |
| `IMessageSourceFeature` | Yes | No | No |
| `IRetryAttemptFeature` | On retry | No | No |
| `IHeadersFeature` | Yes | Yes | No |
| `IRawBytesFeature` | Yes | No | No |
| `IProviderIdentifierFeature` | Yes | Yes | Yes |
| `IResponseFeature` | No | No | Yes |
